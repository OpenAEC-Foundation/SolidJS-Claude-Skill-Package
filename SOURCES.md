## Verification Rules

1. **Primary source**: ALWAYS prefer docs.solidjs.com over any other source
2. **Code verification**: Test code examples against the official SolidJS playground or local project
3. **Version check**: ALWAYS verify which SolidJS version an API belongs to (1.x vs 2.x)
4. **No unofficial sources**: Do NOT use Medium articles, dev.to posts, or Stack Overflow as primary references
5. **GitHub issues**: May be used as secondary source for known bugs and workarounds
6. **SolidStart versioning**: SolidStart has its own version scheme — verify independently from SolidJS core

## Source Quality Tiers

| Tier | Source Type | Usage |
